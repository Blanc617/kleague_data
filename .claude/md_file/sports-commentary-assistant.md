# K리그 스포츠 중계 AI 해설 보조 도구 — 프로젝트 기획서

> 작성일: 2026-03-25
> 대상 리그: K리그1, K리그2

---

## 1. 프로젝트 개요

### 한 줄 소개
> "K리그 해설위원이 경기 중 선수 기록·역대 데이터·흥미로운 통계를 AI에게 실시간으로 질문하면 2~3초 안에 답변을 받는 B2B SaaS"

### 핵심 가치
- 해설위원의 두뇌를 확장하는 도구
- PD·작가가 수백 장 자료를 뒤지던 방식을 AI가 대체
- 예상 밖 선수 교체·이변 상황에서도 즉각 대응 가능

---

## 2. 문제 정의

### 지금 해설 현장의 현실
```
경기 중 갑자기 선수 교체 발생
       ↓
"이 선수 올 시즌 기록이 어떻게 되지?"
       ↓
PD가 옆에서 종이 자료 뒤짐 (30초~1분 소요)
       ↓
이미 다음 장면으로 넘어감 → 타이밍 놓침
```

### 구체적 페인포인트
- 비인기 팀·외국인 선수는 자료 자체가 없는 경우 다수
- 경기 전날 자료 준비에 PD·작가 수 시간 소요
- 이변 상황(부상 교체, 퇴장 등) 즉흥 대응 불가
- 지루한 경기 구간에서 이야깃거리 발굴 어려움

### 이 도구 사용 시
```
경기 중 갑자기 선수 교체 발생
       ↓
해설위원이 태블릿에 타이핑
"김민준 올 시즌 기록 요약해줘"
       ↓
2~3초 안에 스트리밍 답변
"김민준 (전북, FW) — 12경기 출전, 4골 2도움,
 평균 슈팅 2.3회, 최근 3경기 연속 무득점"
       ↓
해설위원이 바로 멘트에 활용
```

## 3. 사용 시나리오

### 경기 전 준비
- 오늘 양 팀 예상 라인업·자료 업로드
- "오늘 주목할 매치업 3가지 뭐야?" 한 번에 정리

### 경기 중 실시간 질의 예시
| 상황 | 질문 | 답변 형태 |
|------|------|----------|
| 선수 교체 직후 | "교체 투입된 이승우 올 시즌 기록" | 수치 요약 3줄 |
| 득점 직후 | "이 선수 올 시즌 몇 번째 골?" | 즉시 수치 반환 |
| 위기 상황 | "이 팀 올 시즌 실점 패턴 어때?" | 패턴 분석 요약 |
| 지루한 구간 | "전북 vs 울산 역대 가장 극적인 경기" | 흥미 유발 스토리 |
| 더비 매치 | "슈퍼매치 최근 5경기 결과 요약" | 전적 테이블 |
| 외국인 선수 | "3번 외국인 선수 이름 발음이랑 출신국" | 발음 가이드 포함 |

## 6. 데이터 소스

### 수집 대상 (K리그1 12팀 + K리그2 13팀)

| 소스 | 수집 데이터 | 방식 |
|------|------------|------|
| kleague.com | 경기 결과, 팀 순위, 공식 발표 | 크롤링 |
| Transfermarkt | 선수 이적 이력, 시장 가치, 부상 기록 | 크롤링 |
| 위키피디아 | 구단 역사, 역대 우승, 유명 선수 커리어 | 자유 사용 |
| 네이버 스포츠 | 인터뷰 기사, 경기 프리뷰·리뷰 | 크롤링 |

### 팀당 수집 데이터 구성
```
├── 선수 명단 (포지션, 나이, 국적, 등번호)
├── 2024·2025 시즌 개인 기록 (득점, 도움, 출전, 경고/퇴장)
├── 팀 경기 결과 (승/무/패, 득실, 홈/원정 분리)
├── 역대 클래식/슈퍼매치 전적 (더비 매치)
└── 주요 선수 인터뷰 기사
```

**총 데이터량**: 텍스트 기준 5~10MB → 임베딩 비용 $1 미만

## 7. 기술 스택
| 레이어 | 기술 | 선택 이유 |
|--------|------|----------|
| 프론트엔드 | react + Typescript + Tailwind CSS | 빠른 프로토타이핑, Vercel 배포 |
| 백엔드 | FastAPI (Python) | LangChain 생태계 통합 용이 |
| RAG 프레임워크 | LangChain | Hybrid Search 지원 |
| 임베딩 | OpenAI text-embedding-3-small | 비용 대비 성능, 한국어 지원 |
| 벡터 DB | Supabase pgvector | 멀티테넌시 격리, DB 통합 관리 |
| LLM | GPT-4o-mini | 비용 최적화, 빠른 응답 |
| 스트리밍 | SSE (Server-Sent Events) | 실시간 저지연 응답 |
| 캐싱 | Redis | 반복 질문 API 비용 절감 |
| 배포 | Vercel (FE) + Railway (BE) | 인프라 관리 최소화 |

Chroma 로컬 단계를 제거하고 Supabase pgvector로 단일화했습니다.

### 핵심 기술: Hybrid Search
```
"2023년 한국시리즈" → 키워드 검색 (BM25) 유리
"극적인 역전 상황"  → 벡터 검색 유리

LangChain EnsembleRetriever
BM25 : 벡터 = 3 : 7 가중치 조합
```
